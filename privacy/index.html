Wayline
How it works Privacy Backend Open planner
Privacy

Clear about what the app sends and what it does not keep.

The current version of Wayline is a trip-planning service, not an account system. It builds itineraries from the trip details you enter and returns the result directly to the browser or desktop app.

What you enter

Origin, destination, driving days, fuel interval, and brand preferences are sent to the planner backend when you build a trip.

What the backend does

The backend uses that request to call Google Routes API and Places API and then returns the finished itinerary to you.

Current behavior

No saved accounts, no built-in trip history.

The current app does not include sign-in, a saved-trips feature, or a user profile. If that changes later, the product copy and privacy page should be updated to match the actual behavior.

Browser exports

PDF, text, and JSON exports are generated for you to save locally.

Hosting logs

Like most web services, the hosting platform may keep standard request logs. If you run Wayline locally or on your own infrastructure, that depends on your environment.

Google Maps data

Trip planning depends on Google Maps Platform responses for routes and places. Their services process the route and search lookups needed to produce the itinerary.
