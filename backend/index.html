Backend

The browser stays light. The routing work happens on the server.

Wayline uses a hosted backend because live trip planning depends on Google Maps web services. That keeps the server key out of the client and lets the planner return one finished itinerary instead of a pile of partial data.

Required APIs

Google Maps Platform

Routes API and Places API (New) are used for route building, gas-stop discovery, and hotel lookups.

Environment

Server-side key only

GOOGLE_MAPS_API_KEY and GOOGLE_MAPS_REGION belong in local environment files or hosted environment variables.

Hosting

Built for Vercel

The current production shape is a Vercel deployment with a static frontend and API routes shipped together.
