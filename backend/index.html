Backend

The live planner needs a backend for one reason: the key stays there.

Wayline’s public-facing UI is static, but live trip planning depends on server-side Google Maps calls. The production web app is structured for Vercel so the frontend and API routes can ship together while the Google API key remains in environment variables.

Quick reference
Required APIs Environment App shape Local preview
Google Cloud

Required APIs

  • Routes API
  • Places API (New)

Those APIs are used for the driving route, gas-stop discovery, and hotel lookups.

Environment variables

Server-side configuration

GOOGLE_MAPS_API_KEY=your_google_maps_key GOOGLE_MAPS_REGION=US

The key belongs in local .env files or host environment variables only. It should never be committed and never shipped in frontend JavaScript.

App shape

Frontend plus API routes

  • / serves the public product site
  • /planner/ serves the planner UI
  • /api/app-state exposes runtime state
  • /api/plan-trip builds the itinerary
Local preview

Useful commands

npm install npm run web npm start

npm run web starts the local web preview with backend routes. npm start opens the desktop app, which uses the same planning engine against your local environment.
